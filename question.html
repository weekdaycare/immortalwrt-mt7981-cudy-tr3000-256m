Q&A ​

luci.cbi.datatypes not found ​

Q: 我在 Openclash 设置中点击更新/下载代理内核时后台管理页面崩溃

A: 使用 ssh 登录路由器后台，win + R 然后输入 cmd 打开命令管理行输入 ssh root@192.168.6.1，然后选 yes ，登录完成之后执行

opkg install luci-compat

推测是依赖的版本兼容问题，暂时想不到解决方法，建议不要手动更新，我会定时编译新固件

128M 装 passwall 代理内核空间不足 ​

由于 128M 分区较小，可能装不上 xray/singbox 这些代理内核，所以我单独编了一版带 passwall 代理内核的

https://www.123865.com/s/xecxjv-LbbCA ，这个固件带 xray 和 singbox 内核

刷第三方 uboot ​

Q: 你的固件能够刷写 mtd/fip 分区吗？

A: 可以，我在编译时集成了 kmod-mtd-rw 插件，刷机需谨慎，小心成砖

USB 供电 ​

Q: 能关闭 USB 供电吗？

A: 可以，ssh 登录路由器后执行

sh
echo 0 > /sys/class/gpio/modem_power/value

恢复供电执行

sh
echo 1 > /sys/class/gpio/modem_power/value
